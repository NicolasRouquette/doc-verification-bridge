    Matrices #

    This file defines basic properties of matrices up to the module structure.

    Matrices with rows indexed by m, columns indexed by n, and entries of type Œ± are represented with Matrix m n Œ±. For the typical approach of counting rows and columns, Matrix (Fin m) (Fin n) Œ± can be used.

    Main definitions #

    • Matrix.transpose: transpose of a matrix, turning rows into columns and vice versa
    • Matrix.submatrix: take a submatrix by reindexing rows and columns
    • Matrix.module: matrices are a module over the ring of entries
    • Set.matrix: set of matrices with entries in a given set

    Notation #

    The scope Matrix gives the following notation:

    • ·µÄ for Matrix.transpose

    See Mathlib/LinearAlgebra/Matrix/ConjTranspose.lean for

    • ·¥¥ for Matrix.conjTranspose

    Implementation notes #

    For convenience, Matrix m n Œ± is defined as m ‚Üí n ‚Üí Œ±, as this allows elements of the matrix to be accessed with A i j. However, it is not advisable to construct matrices using terms of the form fun i j ‚Ü¶ _ or even (fun i j ‚Ü¶ _ : Matrix m n Œ±), as these are not recognized by Lean as having the right type. Instead, Matrix.of should be used.

      (m ‚Üí n ‚Üí Œ±) ‚âÉ Matrix m n Œ±

      Cast a function into a matrix.

      The two sides of the equivalence are definitionally equal types. We want to use an explicit cast to distinguish the types because Matrix has different instances to pi types (such as Pi.mul, which performs elementwise multiplication, vs Matrix.mul).

      If you are defining a matrix, in terms of its entries, use of (fun i j ‚Ü¶ _). The purpose of this approach is to ensure that terms of the form (fun i j ‚Ü¶ _) * (fun i j ‚Ü¶ _) do not appear, as the type of * can be misleading.
