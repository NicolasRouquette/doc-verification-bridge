    Closure operators between preorders #

    We define (bundled) closure operators on a preorder as monotone (increasing), extensive (inflationary) and idempotent functions. We define closed elements for the operator as elements which are fixed by it.

    Lower adjoints to a function between preorders u : Œ≤ ‚Üí Œ± allow to generalise closure operators to situations where the closure operator we are dealing with naturally decomposes as u ‚àò l where l is a worthy function to have on its own. Typical examples include l : Set G ‚Üí Subgroup G := Subgroup.closure, u : Subgroup G ‚Üí Set G := (‚Üë), where G is a group. This shows there is a close connection between closure operators, lower adjoints and Galois connections/insertions: every Galois connection induces a lower adjoint which itself induces a closure operator by composition (see GaloisConnection.lowerAdjoint and LowerAdjoint.closureOperator), and every closure operator on a partial order induces a Galois insertion from the set of closed elements to the underlying type (see ClosureOperator.gi).

    Main definitions #

    • ClosureOperator: A closure operator is a monotone function f : Œ± ‚Üí Œ± such that ‚àÄ x, x ‚â§ f x and ‚àÄ x, f (f x) = f x.
    • LowerAdjoint: A lower adjoint to u : Œ≤ ‚Üí Œ± is a function l : Œ± ‚Üí Œ≤ such that l and u form a Galois connection.

    Implementation details #

    Although LowerAdjoint is technically a generalisation of ClosureOperator (by defining toFun := id), it is desirable to have both as otherwise ids would be carried all over the place when using concrete closure operators such as ConvexHull.

    LowerAdjoint really is a semibundled structure version of GaloisConnection.
