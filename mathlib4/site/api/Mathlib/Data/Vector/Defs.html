    The type List.Vector represents lists with fixed length.

    TODO: The API of List.Vector is quite incomplete relative to Vector, and in particular does not use x[i] (that is GetElem notation) as the preferred accessor. Any combination of reducing the use of List.Vector in Mathlib, or modernising its API, would be welcome.

    List.Vector Œ± n is the type of lists of length n with elements of type Œ±.

    Note that there is also Vector Œ± n in the root namespace, which is the type of arrays of length n with elements of type Œ±.

    Typically, if you are doing programming or verification, you will primarily use Vector Œ± n, and if you are doing mathematics, you may want to use List.Vector Œ± n instead.
