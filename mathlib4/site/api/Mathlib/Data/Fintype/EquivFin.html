    Equivalences between Fintype, Fin and Finite #

    This file defines the bijection between a Fintype Œ± and Fin (Fintype.card Œ±), and uses this to relate Fintype with Finite. From that we can derive properties of Finite and Infinite, and show some instances of Infinite.

    Main declarations #

    • Fintype.truncEquivFin: A fintype Œ± is computably equivalent to Fin (card Œ±). The Trunc-free, noncomputable version is Fintype.equivFin.
    • Fintype.truncEquivOfCardEq Fintype.equivOfCardEq: Two fintypes of same cardinality are equivalent. See above.
    • Fin.equiv_iff_eq: Fin m ‚âÉ Fin n iff m = n.
    • Infinite.natEmbedding: An embedding of ‚Ñï into an infinite type.

    Types which have an injection from/a surjection to an Infinite type are themselves Infinite. See Infinite.of_injective and Infinite.of_surjective.

    Instances #

    We provide Infinite instances for

    • specific types: ‚Ñï, ‚Ñ§, String
    • type constructors: Multiset Œ±, List Œ±
