    Functors which commute with shifts #

    Let C and D be two categories equipped with shifts by an additive monoid A. In this file, we define the notion of functor F : C ‚•§ D which "commutes" with these shifts. The associated type class is [F.CommShift A]. The data consists of commutation isomorphisms F.commShiftIso a : shiftFunctor C a ‚ãô F ‚âÖ F ‚ãô shiftFunctor D a for all a : A which satisfy a compatibility with the addition and the zero. After this was formalised in Lean, it was found that this definition is exactly the definition which appears in Jean-Louis Verdier's thesis (I 1.2.3/1.2.4), although the language is different. (In Verdier's thesis, the shift is not given by a monoidal functor Discrete A ‚•§ C ‚•§ C, but by a fibred category C ‚•§ BA, where BA is the category with one object, the endomorphisms of which identify to A. The choice of a cleavage for this fibered category gives the individual shift functors.)
