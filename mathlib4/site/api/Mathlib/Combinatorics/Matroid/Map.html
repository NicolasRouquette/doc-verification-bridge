    Maps between matroids #

    This file defines maps and comaps, which move a matroid on one type to a matroid on another using a function between the types. The constructions are (up to isomorphism) just combinations of restrictions and parallel extensions, so are not mathematically difficult.

    Because a matroid M : Matroid Œ± is defined with am embedded ground set M.E : Set Œ± which contains all the structure of M, there are several types of map and comap one could reasonably ask for; for instance, we could map M : Matroid Œ± to a Matroid Œ≤ using either a function f : Œ± ‚Üí Œ≤, a function f : ‚ÜëM.E ‚Üí Œ≤ or indeed a function f : ‚ÜëM.E ‚Üí ‚ÜëE for some E : Set Œ≤. We attempt to give definitions that capture most reasonable use cases.

    Matroid.map and Matroid.comap are defined in terms of bare functions rather than functions defined on subtypes, so are often easier to work in practice than the subtype variants. In fact, the statement that N = Matroid.map M f _ for some f : Œ± ‚Üí Œ≤ is equivalent to the existence of an isomorphism from M to N, except in the trivial degenerate case where M is an empty matroid on a nonempty type and N is an empty matroid on an empty type. This can be simpler to use than an actual formal isomorphism, which requires subtypes.

    Main definitions #

    In the definitions below, M and N are matroids on Œ± and Œ≤ respectively.

    • For f : Œ± ‚Üí Œ≤, Matroid.comap N f is the matroid on Œ± with ground set f ‚Åª¬π' N.E in which each I is independent if and only if f is injective on I and f '' I is independent in N. (For each nonloop x of N, the set f ‚Åª¬π' {x} is a parallel class of N.comap f)

    • Matroid.comapOn N f E is the restriction of N.comap f to E for some E : Set Œ±.

    • For an embedding f : M.E ‚Ü™ Œ≤ defined on the subtype ‚ÜëM.E, Matroid.mapSetEmbedding M f is the matroid on Œ≤ with ground set range f whose independent sets are the images of those in M. This matroid is isomorphic to M.

    • For a function f : Œ± ‚Üí Œ≤ and a proof hf that f is injective on M.E, Matroid.map f hf is the matroid on Œ≤ with ground set f '' M.E whose independent sets are the images of those in M. This matroid is isomorphic to M, and does not depend on the values f takes outside M.E.

    • Matroid.mapEmbedding f is a version of Matroid.map where f : Œ± ‚Ü™ Œ≤ is a bundled embedding. It is defined separately because the global injectivity of f gives some nicer simp lemmas.

    • Matroid.mapEquiv f is a version of Matroid.map where f : Œ± ‚âÉ Œ≤ is a bundled equivalence. It is defined separately because we get even nicer simp lemmas.

    • Matroid.mapSetEquiv f is a version of Matroid.map where f : M.E ‚âÉ E is an equivalence on subtypes. It gives a matroid on Œ≤ with ground set E.

    • For X : Set Œ±, Matroid.restrictSubtype M X is the Matroid ‚Ü•X with ground set univ : Set ‚Ü•X. This matroid is isomorphic to M ‚Üæ X.

    Implementation details #

    The definition of comap is the only place where we need to actually define a matroid from scratch. After comap is defined, we can define map and its variants indirectly in terms of comap.

    If f : Œ± ‚Üí Œ≤ is injective on M.E, the independent sets of M.map f hf are the images of the independent set of M; i.e. (M.map f hf).Indep I ‚Üî ‚àÉ I‚ÇÄ, M.Indep I‚ÇÄ ‚àß I = f '' I‚ÇÄ. But if f is globally injective, we can phrase this more directly; indeed, (M.map f _).Indep I ‚Üî M.Indep (f ‚Åª¬π' I) ‚àß I ‚äÜ range f. If f is an equivalence we have (M.map f _).Indep I ‚Üî M.Indep (f.symm '' I). In order that these stronger statements can be @[simp], we define mapEmbedding and mapEquiv separately from map.

    Notes #

    For finite matroids, both maps and comaps are a special case of a construction of Perfect [perfect1969matroid] in which a matroid structure can be transported across an arbitrary bipartite graph that may not correspond to a function at all (See [oxley2011], Theorem 11.2.12). It would have been nice to use this more general construction as a basis for the definition of both Matroid.map and Matroid.comap.

    Unfortunately, we can't do this, because the construction doesn't extend to infinite matroids. Specifically, if M‚ÇÅ and M‚ÇÇ are matroids on the same type Œ±, and f is the natural function from Œ± ‚äï Œ± to Œ±, then the images under f of the independent sets of the direct sum M‚ÇÅ ‚äï M‚ÇÇ are the independent sets of a matroid if and only if the union of M‚ÇÅ and M‚ÇÇ is a matroid, and unions do not exist for some pairs of infinite matroids: see [aignerhorev2012infinite]. For this reason, Matroid.map requires injectivity to be well-defined in general.
