    Encodable types #

    This file defines encodable (constructively countable) types as a typeclass. This is used to provide explicit encode/decode functions from and to ‚Ñï, with the information that those functions are inverses of each other. The difference with Denumerable is that finite types are encodable. For infinite types, Encodable and Denumerable agree.

    Main declarations #

    • Encodable Œ±: States that there exists an explicit encoding function encode : Œ± ‚Üí ‚Ñï with a partial inverse decode : ‚Ñï ‚Üí Option Œ±.
    • decode‚ÇÇ: Version of decode that is equal to none outside of the range of encode. Useful as we do not require this in the definition of decode.
    • ULower Œ±: Any encodable type has an equivalent type living in the lowest universe, namely a subtype of ‚Ñï. ULower Œ± finds it.

    Implementation notes #

    The point of asking for an explicit partial inverse decode : ‚Ñï ‚Üí Option Œ± to encode : Œ± ‚Üí ‚Ñï is to make the range of encode decidable even when the finiteness of Œ± is not.
