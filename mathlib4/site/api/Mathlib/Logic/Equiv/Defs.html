    Equivalence between types #

    In this file we define two types:

    • Equiv Œ± Œ≤ a.k.a. Œ± ‚âÉ Œ≤: a bijective map Œ± ‚Üí Œ≤ bundled with its inverse map; we use this (and not equality!) to express that various Types or Sorts are equivalent.

    • Equiv.Perm Œ±: the group of permutations Œ± ‚âÉ Œ±. More lemmas about Equiv.Perm can be found in Mathlib/GroupTheory/Perm/.

    Then we define

    • canonical isomorphisms between various types: e.g.,

      • Equiv.refl Œ± is the identity map interpreted as Œ± ‚âÉ Œ±;

    • operations on equivalences: e.g.,

      • Equiv.symm e : Œ≤ ‚âÉ Œ± is the inverse of e : Œ± ‚âÉ Œ≤;

      • Equiv.trans e‚ÇÅ e‚ÇÇ : Œ± ‚âÉ Œ≥ is the composition of e‚ÇÅ : Œ± ‚âÉ Œ≤ and e‚ÇÇ : Œ≤ ‚âÉ Œ≥ (note the order of the arguments!);

    • definitions that transfer some instances along an equivalence. By convention, we transfer instances from right to left.

      • Equiv.inhabited takes e : Œ± ‚âÉ Œ≤ and [Inhabited Œ≤] and returns Inhabited Œ±;
      • Equiv.unique takes e : Œ± ‚âÉ Œ≤ and [Unique Œ≤] and returns Unique Œ±;
      • Equiv.decidableEq takes e : Œ± ‚âÉ Œ≤ and [DecidableEq Œ≤] and returns DecidableEq Œ±.

      More definitions of this kind can be found in other files. E.g., Mathlib/Algebra/Group/TransferInstance.lean does it for Group, Mathlib/Algebra/Module/TransferInstance.lean does it for Module, and similar files exist for other algebraic type classes.

    Many more such isomorphisms and operations are defined in Mathlib/Logic/Equiv/Basic.lean.
