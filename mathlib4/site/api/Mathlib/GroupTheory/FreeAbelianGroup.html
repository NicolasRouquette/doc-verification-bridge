    Free abelian groups #

    The free abelian group on a type Œ±, defined as the abelianisation of the free group on Œ±.

    The free abelian group on Œ± can be abstractly defined as the left adjoint of the forgetful functor from abelian groups to types. Alternatively, one could define it as the functions Œ± ‚Üí ‚Ñ§ which send all but finitely many (a : Œ±) to 0, under pointwise addition. In this file, it is defined as the abelianisation of the free group on Œ±. All the constructions and theorems required to show the adjointness of the construction and the forgetful functor are proved in this file, but the category-theoretic adjunction statement is in Mathlib/Algebra/Category/Grp/Adjunctions.lean.

    Main definitions #

    Here we use the following variables: (Œ± Œ≤ : Type*) (A : Type*) [AddCommGroup A]

    • FreeAbelianGroup Œ± : the free abelian group on a type Œ±. As an abelian group it is Œ± ‚Üí‚ÇÄ ‚Ñ§, the functions from Œ± to ‚Ñ§ such that all but finitely many elements get mapped to zero, however this is not how it is implemented.

    • lift f : FreeAbelianGroup Œ± ‚Üí+ A : the group homomorphism induced by the map f : Œ± ‚Üí A.

    • map (f : Œ± ‚Üí Œ≤) : FreeAbelianGroup Œ± ‚Üí+ FreeAbelianGroup Œ≤ : functoriality of FreeAbelianGroup.

    • instance [Monoid Œ±] : Semigroup (FreeAbelianGroup Œ±)

    • instance [CommMonoid Œ±] : CommRing (FreeAbelianGroup Œ±)

    It has been suggested that we would be better off refactoring this file and using Finsupp instead.

    Implementation issues #

    The definition is def FreeAbelianGroup : Type u := Additive <| Abelianization <| FreeGroup Œ±.

    Chris Hughes has suggested that this all be rewritten in terms of Finsupp. Johan Commelin has written all the API relating the definition to Finsupp in the lean-liquid repo.

    The lemmas map_pure, map_of, map_zero, map_add, map_neg and map_sub are proved about the Functor.map <$> construction, and need Œ± and Œ≤ to be in the same universe. But FreeAbelianGroup.map (f : Œ± ‚Üí Œ≤) is defined to be the AddGroup homomorphism FreeAbelianGroup Œ± ‚Üí+ FreeAbelianGroup Œ≤ (with Œ± and Œ≤ now allowed to be in different universes), so (map f).map_add etc. can be used to prove that FreeAbelianGroup.map preserves addition. The functions map_id, map_id_apply, map_comp, map_comp_apply and map_of_apply are about FreeAbelianGroup.map.

    def FreeAbelianGroup (Œ± : Type u) :
    Type u

    If Œ± is a type, then FreeAbelianGroup Œ± is the free abelian group generated by Œ±. This is an abelian group equipped with a function FreeAbelianGroup.of : Œ± ‚Üí FreeAbelianGroup Œ± which has the following universal property: if G is any abelian group, and f : Œ± ‚Üí G is any function, then this function is the composite of FreeAbelianGroup.of and a unique group homomorphism FreeAbelianGroup.lift f : FreeAbelianGroup Œ± ‚Üí+ G.

    A typical element of FreeAbelianGroup Œ± is a formal sum of elements of Œ± and their formal inverses. For example if x and y are terms of type Œ± then x + x + x - y is a "typical" element of FreeAbelianGroup Œ±. In particular if Œ± is empty then FreeAbelianGroup Œ± is isomorphic to the trivial group, and if Œ± has one term then FreeAbelianGroup Œ± is isomorphic to ‚Ñ§. One can think of FreeAbelianGroup Œ± as the functions Œ± ‚Üí‚ÇÄ ‚Ñ§ with finite support, and addition given pointwise.
