    Matroid Restriction #

    Given M : Matroid Î± and R : Set Î±, the independent sets of M that are contained in R are the independent sets of another matroid M â†¾ R with ground set R, called the 'restriction' of M to R.     • M.restrict R, written M â†¾ R, is the restriction of M : Matroid Î± to R : Set Î±: i.e. the matroid with ground set R whose independent sets are the M-independent subsets of R.

    • Matroid.Restriction N M, written N â‰¤r M, means that N = M â†¾ R for some R âŠ† M.E.

    • Matroid.IsStrictRestriction N M, written N <r M, means that N = M â†¾ R for some R âŠ‚ M.E.

    • Matroidáµ£ Î± is a type synonym for Matroid Î±, equipped with the PartialOrder â‰¤r.

    Implementation Notes #

    Since R and M.E are both terms in Set Î±, to define the restriction M â†¾ R, we need to either insist that R âŠ† M.E, or to say what happens when R contains the junk outside M.E.

    It turns out that R âŠ† M.E is just an unnecessary hypothesis; if we say the restriction M â†¾ R has ground set R and its independent sets are the M-independent subsets of R, we always get a matroid, in which the elements of R \ M.E aren't in any independent sets. We could instead define this matroid to always be 'smaller' than M by setting (M â†¾ R).E := R âˆ© M.E, but this is worse definitionally, and more generally less convenient.

    This makes it possible to actually restrict a matroid 'upwards'; for instance, if M : Matroid Î± satisfies M.E = âˆ…, then M â†¾ Set.univ is the matroid on Î± whose ground set is all of Î±, where the empty set is the only independent set. (In general, elements of R \ M.E are all 'loops' of the matroid M â†¾ R; see Matroid.loops and Matroid.restrict_loops_eq' for a precise version of this statement.) This is mathematically strange, but is useful for API building.

    The cost of allowing a restriction of M to be 'bigger' than M itself is that the statement M â†¾ R â‰¤r M is only true with the hypothesis R âŠ† M.E (at least, if we want â‰¤r to be a partial order). But this isn't too inconvenient in practice. Indeed (Â· âŠ† M.E) proofs can often be automatically provided by aesop_mat.

    We define the restriction order â‰¤r to give a PartialOrder instance on the type synonym Matroidáµ£ Î± rather than Matroid Î± itself, because the PartialOrder (Matroid Î±) instance is reserved for the more mathematically important 'minor' order; see Matroid.IsMinor.
