    Multiplicative opposite and algebraic operations on it #

    In this file we define MulOpposite Œ± = Œ±·µê·µí·µñ to be the multiplicative opposite of Œ±. It inherits all additive algebraic structures on Œ± (in other files), and reverses the order of multipliers in multiplicative structures, i.e., op (x * y) = op y * op x, where MulOpposite.op is the canonical map from Œ± to Œ±·µê·µí·µñ.

    We also define AddOpposite Œ± = Œ±·µÉ·µí·µñ to be the additive opposite of Œ±. It inherits all multiplicative algebraic structures on Œ± (in other files), and reverses the order of summands in additive structures, i.e. op (x + y) = op y + op x, where AddOpposite.op is the canonical map from Œ± to Œ±·µÉ·µí·µñ.

    Notation #

    • Œ±·µê·µí·µñ = MulOpposite Œ±
    • Œ±·µÉ·µí·µñ = AddOpposite Œ±

    Implementation notes #

    In mathlib3 Œ±·µê·µí·µñ was just a type synonym for Œ±, marked irreducible after the API was developed. In mathlib4 we use a structure with one field, because it is not possible to change the reducibility of a declaration after its definition, and because Lean 4 has definitional eta reduction for structures (Lean 3 does not).

    structure PreOpposite (Œ± : Type u_3) :
    Type u_3

    Auxiliary type to implement MulOpposite and AddOpposite.

    It turns out to be convenient to have MulOpposite Œ± = AddOpposite Œ± true by definition, in the same way that it is convenient to have Additive Œ± = Œ±; this means that we also get the defeq AddOpposite (Additive Œ±) = MulOpposite Œ±, which is convenient when working with quotients.

    This is a compromise between making MulOpposite Œ± = AddOpposite Œ± = Œ± (what we had in Lean 3) and having no defeqs within those three types (which we had as of https://github.com/leanprover-community/mathlib4/pull/1036).
