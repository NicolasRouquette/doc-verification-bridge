    Multiset coercion to type #

    This module defines a CoeSort instance for multisets and gives it a Fintype instance. It also defines Multiset.toEnumFinset, which is another way to enumerate the elements of a multiset. These coercions and definitions make it easier to sum over multisets using existing Finset theory.

    Main definitions #

    • A coercion from m : Multiset Î± to a Type*. Each x : m has two components. The first, x.1, can be obtained via the coercion â†‘x : Î±, and it yields the underlying element of the multiset. The second, x.2, is a term of Fin (m.count x), and its function is to ensure each term appears with the correct multiplicity. Note that this coercion requires DecidableEq Î± due to the definition using Multiset.count.
    • Multiset.toEnumFinset is a Finset version of this.
    • Multiset.coeEmbedding is the embedding m â†ª Î± Ã— â„•, whose first component is the coercion and whose second component enumerates elements with multiplicity.
    • Multiset.coeEquiv is the equivalence m â‰ƒ m.toEnumFinset.
