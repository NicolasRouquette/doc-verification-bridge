    Krull dimension of a preordered set and height of an element #

    If Œ± is a preordered set, then krullDim Œ± : WithBot ‚Ñï‚àû is defined to be sup {n | a‚ÇÄ < a‚ÇÅ < ... < a‚Çô}.

    In case that Œ± is empty, then its Krull dimension is defined to be negative infinity; if the length of all series a‚ÇÄ < a‚ÇÅ < ... < a‚Çô is unbounded, then its Krull dimension is defined to be positive infinity.

    For a : Œ±, its height (in ‚Ñï‚àû) is defined to be sup {n | a‚ÇÄ < a‚ÇÅ < ... < a‚Çô ‚â§ a}, while its coheight is defined to be sup {n | a ‚â§ a‚ÇÄ < a‚ÇÅ < ... < a‚Çô} .

    Main results #

    • The Krull dimension is the same as that of the dual order (krullDim_orderDual).

    • The Krull dimension is the supremum of the heights of the elements (krullDim_eq_iSup_height), or their coheights (krullDim_eq_iSup_coheight), or their sums of height and coheight (krullDim_eq_iSup_height_add_coheight_of_nonempty)

    • The height in the dual order equals the coheight, and vice versa.

    • The height is monotone (height_mono), and strictly monotone if finite (height_strictMono).

    • The coheight is antitone (coheight_anti), and strictly antitone if finite (coheight_strictAnti).

    • The height is the supremum of the successor of the height of all smaller elements (height_eq_iSup_lt_height).

    • The elements of height zero are the minimal elements (height_eq_zero), and the elements of height n are minimal among those of height ‚â• n (height_eq_coe_iff_minimal_le_height).

    • Concrete calculations for the height, coheight and Krull dimension in ‚Ñï, ‚Ñ§, WithTop, WithBot and ‚Ñï‚àû.

    Design notes #

    Krull dimensions are defined to take value in WithBot ‚Ñï‚àû so that (-‚àû) + (+‚àû) is also negative infinity. This is because we want Krull dimensions to be additive with respect to product of varieties so that -‚àû being the Krull dimension of empty variety is equal to sum of -‚àû and the Krull dimension of any other varieties.

    We could generalize the notion of Krull dimension to an arbitrary binary relation; many results in this file would generalize as well. But we don't think it would be useful, so we only define Krull dimension of a preorder.
