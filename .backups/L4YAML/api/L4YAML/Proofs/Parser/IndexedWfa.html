    IndexedWfa — Phase 3 Step 6d.2 indexed WFA proofs (staging) #

    Status: staging file. Not imported by L4YAML.lean until the Phase 3 Step 6f cutover commit.

    Role #

    Indexed twin of L4YAML/Proofs/Parser/ParserWfaProofs.lean: the WellFormedAnchors preservation proofs reparented onto ParseStateIx input and the indexed parseNode / parseDocument / parseStreamIx family in L4YAML.TokenParser.Indexed.

    Architecture mirrors the legacy file (strong induction on fuel), substituting state type and consuming the indexed WellBehaved substrate (parseNode_wb_all_ix, parseNodeContent_wb_ix, parseNodeProperties_tokens_ix) directly. Only parseNodeProperties needs a new _anchors_eq_ix companion since the legacy proof was written for ParseState-shaped tokens; the body is a near-verbatim copy with ParseStateIx.advance substitutions.

    Phase 3 Step 6f cutover #

    At cutover, this file is renamed to Proofs/Parser/ParserWfaProofs.lean (overwriting the legacy file) and the namespace L4YAML.Proofs.Indexed.WfaProofs reverts to L4YAML.Proofs.ParserWfaProofs.
