    List/String Algebra (Algebra Items 9 + 22) #

    Foundational list/string decomposition algebra used by whitespace handling, scalar lexing, and character-class predicates in the YAML parser.

    • Item 9 — character/string decomposition. String.toList, ++, prefix/suffix laws. Re-states the core List monoid laws under the algebra namespace, plus a few String ↔ List Char bridge lemmas that recur throughout the parser/scanner.
    • Item 22(a) — dropWhile_idempotent: dropping the same prefix twice = once.
    • Item 22(b) — reverse_dropWhile_reverse_idempotent: the reverse-trim-reverse pattern is idempotent.

    Closure (Guardrail 2) #

    The Item 9 lemmas here are all (a) free-monoid laws on List / String, or (b) the String.toList / String.length / String.append bridge equations that String instances satisfy in core Lean. No content beyond the algebra inventory is added.

    Provenance #

    Item 22 lemmas migrated from L4YAML/Proofs/Foundation/StringProperties.lean:71–91 during Initiative 4 Phase 2 (D4: one file per item-cluster). Item 9 lemmas are new content (re-stating core Lean laws under this file's namespace so the algebra layer is self-describing).

    The remainder of StringProperties.lean (FoldResult, validPlainFirst preservation, etc.) is not part of the algebra inventory and stays in Proofs/Foundation/ for now.

    Item 9 — Prefix / suffix decomposition #

    `xs <+: ys ↔ ∃ zs, ys = xs ++ zs` is the canonical
decomposition lemma. `List.prefix_iff_eq_append` (core Lean)
provides this; we re-state it here so the algebra layer is the
one-stop reference.
    source
    theorem L4YAML.Algebra.StringList.prefix_iff_exists_append {α : Type u_1} {xs ys : List α} :
    xs <+: ys ↔ ∃ (zs : List α), ys = xs ++ zs

    A prefix decomposes the underlying list as xs ++ zs.

    source
    theorem L4YAML.Algebra.StringList.prefix_append_right {α : Type u_1} (xs ys : List α) :
    xs <+: xs ++ ys

    Every list is a prefix of itself appended to anything.

    Item 9 — String ↔ List Char bridge #

    source
    theorem L4YAML.Algebra.StringList.toList_append (s t : String) :
    (s ++ t).toList = s.toList ++ t.toList

    String append distributes over toList: (s ++ t).toList = s.toList ++ t.toList. (Re-export of core Lean's String.toList_append.)

    source
    theorem L4YAML.Algebra.StringList.length_append (s t : String) :
    (s ++ t).length = s.length + t.length

    String length is list length: (s ++ t).length = s.length + t.length.

    Item 22 — dropWhile idempotence #

    source
    theorem L4YAML.Algebra.StringList.dropWhile_cons_false {α : Type} (p : α → Bool) (x : α) (xs : List α) (h : p x = false) :
    List.dropWhile p (x :: xs) = x :: xs

    dropWhile p on a list starting with an element where p is false returns the entire list.

    source
    theorem L4YAML.Algebra.StringList.dropWhile_idempotent {α : Type} (p : α → Bool) (xs : List α) :
    List.dropWhile p (List.dropWhile p xs) = List.dropWhile p xs

    Item 22(a) — list-level idempotence: After dropping elements matching p, applying dropWhile p again is a no-op.

    source
    theorem L4YAML.Algebra.StringList.reverse_dropWhile_reverse_idempotent {α : Type} (p : α → Bool) (xs : List α) :
    (List.dropWhile p (List.dropWhile p xs.reverse).reverse.reverse).reverse = (List.dropWhile p xs.reverse).reverse

    Item 22(b) — reverse-trim-reverse idempotent: Applying the full trim operation twice is the same as once, at the list level.
