                                            def «command_#dualize» :
                                            Lean.TrailingParserDescr

                                            This is a command to reduce the repetitive work of proving things twice, once for homology and again for cohomology.

                                            Writing #dualize beneath a theorem/lemma/structure/class offers a pop-up that suggests a dual version where all the names involving "cohomology" have been swapped with "homology" and vice-versa.
