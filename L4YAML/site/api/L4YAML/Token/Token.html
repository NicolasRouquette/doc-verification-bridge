    YAML Token Types #

    Explicit token types for the two-pass scanner/parser architecture (Phase 9).

    YAML 1.2.2 defines 205 productions in three layers:

    • Character class (C, 18 productions): predicates on Char — not tokens
    • Lexical/Token (L, 132 productions): character patterns → tokens
    • Syntactic/Grammar (S, 54 productions): tokens → AST node

    The YamlToken type represents the output of the L-layer scanner and the input to the S-layer grammar parser. See YAML_PRODUCTIONS.md §Token–Grammar Layer Analysis for the full production classification.

    Design Decisions #

    1. Reuse ScalarStyle from Types.lean rather than defining a new enum. The five styles (plain, single-quoted, double-quoted, literal, folded) are shared between scanner output and AST.

    2. Resolved scalar content. The scalar token carries fully-resolved text: escape sequences expanded, line folding applied, chomp style applied. All of productions [41]–[61] (escapes), [69]–[72] (folding), [106]–[132] (scalar content rules), and [158]–[175] (block scalar processing) are internal to the scanner's scalar tokenization logic.

    3. Virtual tokens. blockSequenceStart, blockMappingStart, and blockEnd have no character representation in the input. They are generated by the scanner tracking an indentation stack, analogous to Python's INDENT/DEDENT.

    4. Position tracking. Each token carries its source position (YamlPos) for error reporting and proof correspondence.

    structure L4YAML.Positioned (α : Type) :
    Type

    A positioned token: token value paired with its source location.

    The start position records where the token starts in the input — used for error messages and for proof correspondence between the scanner and the original character stream. The optional end position marks the first byte after the last character consumed by the token (exclusive).

    • pos : YamlPos

      Source position where this token starts

    • val : α

      The token value

    • endPos : YamlPos

      Source position just after the last character of this token (exclusive). Populated by the scanner at every emission site. For virtual tokens (blockEnd, blockSequenceStart, blockMappingStart, placeholder) this equals pos (zero-width).

              inductive L4YAML.YamlToken :
              Type

              YAML 1.2.2 token types.

              Based on the layer analysis of all 205 spec productions. Follows the libyaml scanner model (yaml_token_type_e).

              Productions [1]–[40] (character classes) are predicates on Char, not tokens. Productions [41]–[200] (lexical) produce these tokens. Productions [94], [103]–[105], [133]–[157], [176]–[195], [201]–[205] (syntactic) compose tokens into the AST.

              • streamStart : YamlToken

                Beginning of input. Always the first token emitted.

              • streamEnd : YamlToken

                End of input. Always the last token emitted.

              • placeholder : YamlToken

                Reserved slot for potential simple key resolution. Pushed by saveSimpleKey to pre-allocate space for .key and optionally .blockMappingStart. Overwritten in-place by scanValuePrepare when the key is confirmed, or filtered out by scan before returning the final token array. Never appears in the output token stream.

              • versionDirective (major minor : Nat) : YamlToken

                %YAML major.minor version directive.

              • tagDirective (handle tagPrefix : String) : YamlToken

                %TAG !handle! prefix tag shorthand directive.

              • documentStart : YamlToken

                --- document start marker.

              • documentEnd : YamlToken

                ... document end marker.

              • blockSequenceStart : YamlToken

                Indentation increase before - (block sequence entry). Generated when indentation exceeds the current level and the next non-space character is -.

              • blockMappingStart : YamlToken

                Indentation increase before a mapping key. Generated when indentation exceeds the current level and the context indicates a block mapping.

              • blockEnd : YamlToken

                Indentation decrease back to a previous level. Generated for each level popped from the indentation stack.

              • blockEntry : YamlToken

                - followed by whitespace or break (block sequence entry indicator). YAML 1.2.2: [4] c-sequence-entry

              • key : YamlToken

                ? followed by whitespace or break (explicit mapping key indicator). YAML 1.2.2: [5] c-mapping-key

              • value : YamlToken

                : followed by whitespace or break (mapping value indicator). YAML 1.2.2: [6] c-mapping-value

              • flowSequenceStart : YamlToken

                [ flow sequence start. YAML 1.2.2: [8] c-sequence-start

              • flowSequenceEnd : YamlToken

                ] flow sequence end. YAML 1.2.2: [9] c-sequence-end

              • flowMappingStart : YamlToken

                { flow mapping start. YAML 1.2.2: [10] c-mapping-start

              • flowMappingEnd : YamlToken

                } flow mapping end. YAML 1.2.2: [11] c-mapping-end

              • flowEntry : YamlToken

                , flow entry separator. YAML 1.2.2: [7] c-collect-entry

              • anchor (name : String) : YamlToken

                &name anchor definition. YAML 1.2.2: [101] c-ns-anchor-property

              • alias (name : String) : YamlToken

                *name alias reference. YAML 1.2.2: [104] c-ns-alias-node

              • tag (handle suffix : String) : YamlToken

                Tag: !suffix, !!suffix, !handle!suffix, or !<uri>. YAML 1.2.2: [97]–[100] c-ns-tag-property

              • scalar (value : String) (style : ScalarStyle) : YamlToken

                Scalar content with its presentation style. The value is the fully-resolved content string (escapes expanded, lines folded per style rules, chomp applied for block scalars). The style records how it appeared in the source for round-trip.

              • comment (text : String) : YamlToken

                Comment text (excluding the leading # and whitespace). Preserved for Phase 8 (comment round-trip fidelity).

                            inductive L4YAML.ScanError :
                            Type

                            Structured error type for the YAML scanner and grammar parser.

                            Replaces unstructured Except String throughout Scanner.lean and TokenParser.lean (P10.6e.2). Each constructor corresponds to a specific error category, making errors machine-inspectable and pattern-matchable.

                            Scanner errors (13 sites in Scanner.lean): character-level violations detected during tokenization.

                            Parser errors (5 sites in TokenParser.lean): token-level violations detected during grammar parsing.

                            • tabInIndentation (line col : Nat) : ScanError

                              Tab character used as indentation — §6.1 violation. s-indent(n) [63] requires spaces only.

                            • unexpectedChar (c : Char) (line col : Nat) : ScanError

                              Character cannot start any token in the current context.

                            • unterminatedScalar (style : ScalarStyle) (line : Nat) : ScanError

                              Quoted scalar opened but never closed (missing closing " or ').

                            • unterminatedEscape (line : Nat) : ScanError

                              Escape sequence \ at end of input with no following character.

                            • unknownEscape (c : Char) (line : Nat) : ScanError

                              Backslash followed by an unrecognized escape character.

                            • invalidHexEscape (expected found line : Nat) : ScanError

                              Hex escape (\x, \u, \U) has fewer hex digits than required.

                            • unicodeOutOfRange (line : Nat) : ScanError

                              Hex escape value exceeds Unicode range (> U+10FFFF).

                            • expectedNewline (line : Nat) : ScanError

                              Block scalar header not followed by a newline.

                            • fuelExhausted (line col : Nat) : ScanError

                              Scanner iteration limit reached (defensive guard).

                            • directiveTrailingContent (line col : Nat) : ScanError

                              Trailing content after %YAML version — §6.8.1 violation.

                            • duplicateYamlDirective (line : Nat) : ScanError

                              Duplicate %YAML directive in the same document — §6.8.1 violation.

                            • directiveAfterContent (line : Nat) : ScanError

                              %YAML/%TAG directive after document content without ... — §6.8 violation.

                            • directiveWithoutDocument (line : Nat) : ScanError

                              Directives present but no document follows (--- required) — §6.8 violation.

                            • documentMarkerInScalar (style : ScalarStyle) (line : Nat) : ScanError

                              Document marker (---/...) at col 0 terminates multi-line quoted scalar — §9.1.2.

                            • trailingContentAfterDocEnd (line col : Nat) : ScanError

                              Non-comment content after document-end marker ... — §9.1.2 violation.

                            • flowEndOutsideFlow (bracket : Char) (line col : Nat) : ScanError

                              ] or } encountered outside any flow collection — §7.4 violation.

                            • underIndentedScalar (style : ScalarStyle) (line : Nat) : ScanError

                              Continuation line of quoted scalar is under-indented — §8.1 violation.

                            • documentMarkerInFlow (line : Nat) : ScanError

                              Document marker (---/...) inside flow collection — §5.4 violation.

                            • underIndentedFlowContent (line col : Nat) : ScanError

                              Flow content below block indentation level — §8.1 violation.

                            • invalidControlChar (c : Char) (style : ScalarStyle) (line col : Nat) : ScanError

                              C0 control character (not matching [2] nb-json) inside quoted scalar — §5.1 violation.

                            • expectedToken (desc : String) (line : Nat) (got : Option String) : ScanError

                              Expected a specific token but found something else (or end of input).

                            • nestingDepthExceeded (line : Nat) : ScanError

                              Recursive nesting depth exceeded maxDepth (stack overflow guard).

                            • multipleDocuments (count : Nat) : ScanError

                              parseYamlSingle* called but input contains multiple documents.

                            • trailingContent (line col : Nat) : ScanError

                              Unexpected content after a node where no more tokens are expected. E.g., trailing text after a quoted scalar in a block mapping value, or content after a flow collection close in block context.

                            • duplicateAnchor (line : Nat) : ScanError

                              Anchor defined twice on the same node — §6.9.2 violation.

                            • contentOnDocumentStartLine (line col : Nat) : ScanError

                              Unexpected content on the --- document-start line. YAML §9.1.1: document start is --- followed by s-l-comments or content on a new line. Content on the same line as --- without directives context is ambiguous.

                            • unterminatedFlowCollection (bracket : Char) (line : Nat) : ScanError

                              Flow collection opened but never closed (missing ] or }).

                            • invalidFlowEntry (line col : Nat) : ScanError

                              Comma at an invalid position in a flow collection (leading or duplicate).

                            • invalidImplicitKey (line : Nat) : ScanError

                              Implicit key used as value indicator without preceding key content — §7.4 violation.

                            • invalidBareDocument (line col : Nat) : ScanError

                              Bare document (no ---) after a previous document without ... — §9.2 [211] violation. YAML streams allow bare documents only as the first document or after ....

                            • undeclaredTagHandle (handle : String) (line col : Nat) : ScanError

                              Tag shorthand handle used without a corresponding %TAG directive — §6.8.2.2 violation. Named handles (e.g., !prefix!) are local to the document and must be declared.

                            • blockScalarIndentMismatch (line col : Nat) : ScanError

                              Whitespace-only line in block scalar exceeds detected content indent — §8.1.3.

                            • undefinedAlias (name : String) (line col : Nat) : ScanError

                              Alias *name used without a preceding &name anchor in this document — §7.1 violation.

                            • misindentedExplicitValue (line col : Nat) (expected : Int) : ScanError

                              Value indicator : after explicit key ? at wrong indentation — §8.2.2 [197] requires s-indent(n) before : in l-block-map-explicit-value(n).

                            • sameLineExplicitValue (line col : Nat) : ScanError

                              Value indicator : on same line as ? with no preceding implicit key — §8.2.2 [197]: l-block-map-explicit-value(n) has l- prefix (line-start); : as explicit value must be on its own line.

                            • emptyAnchorName (line col : Nat) : ScanError

                              Anchor & or alias * followed by no name characters — §6.9 violation. ns-anchor-name [103] requires at least one ns-anchor-char [102].

                            • emptyVerbatimTagURI (line col : Nat) : ScanError

                              Verbatim tag !<...> with empty URI — §6.9 [98] violation. c-verbatim-tag requires at least one ns-uri-char [39].

                            • unterminatedVerbatimTag (line col : Nat) : ScanError

                              Verbatim tag !<... missing closing > — §6.9 [98] violation.
