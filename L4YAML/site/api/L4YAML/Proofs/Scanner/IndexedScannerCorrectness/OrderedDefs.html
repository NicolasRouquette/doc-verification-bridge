              §8.2' Position-preserving mono helpers. #

              The _mono family above requires full token equality on the prefix (s'.tokens[i] = s.tokens[i]). The _mono_pos family weakens that to .start-only equality — sufficient for SimpleKeyValidIx / SimpleKeyStackValidIx, which only mention .start, and necessary for overwriteAtCursor-chained helpers like scanValuePrepareIx (which do change the token's .kind field while preserving .start).

              Indexed twin of legacy SimpleKeyStackValid_mono_pos (ScannerCorrectness.lean:8803).
